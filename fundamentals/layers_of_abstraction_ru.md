# Уровни абстракции
Из [страницы про абстракции](abstraction_ru.md) мы узнали, что это такое и как они формируются. Здесь же мы посмотрим, почему у одного и того же объекта могут быть выделены разные абстракции.

## Описание

Вернёмся к фрагменту кода на псевдоязыке из прошлого документа:
```
index = 0
sum = 0
elements = [1, 2, 3, 4]
while index < elements.size {
  sum += elements(index)
  index++
}
```

В этом фрагменте кода мы выделили такие абстракции:
```
index = 0 <- переменная, присваивание, число
sum = 0
elements = [1, 2, 3, 4] <- массив (коллекция, кортеж, ..)
while index < elements.size { <- цикл, условие, размер массива
  sum += elements(index) <- доступ к элементу массива, присваивание со сложением
  index += 1
}
```

Можно же выделить и более "общие" абстракции:
```
index = 0 <- переменная, операция, значение
sum = 0
elements = [1, 2, 3, 4]
while index < elements.size { <- цикл, условие, поле объекта
  sum += elements(index) <- функция объекта
  index += 1
}
```

Теперь у нас нет конкретных видов операций в коде, но есть само понятие "операции", которая происходит в данном коде. То же самое относится и к массиву.

Можно выделить и более "конкретные" абстракции:
```
index = 0 <- регистр, запись в регистр, байтовое значение
sum = 0
elements = [1, 2, 3, 4] <- последовательность байтов
while index < elements.size { <- метка, операция сравнения, чтение регистра
  sum += elements(index) <- стек вызовов
  index += 1
}
```
Здесь уже абстракции выходят сильно конкретней, с предположением, что код будет исполняться на уровне команд языков ассемблера т.е. команд общения с процессором.

Отсюда мы видим, что у одного и того же кода абстракции в нём могут быть разными, в зависимости от перспективы взгляда на него. Это называется **уровнем абстракции**. Смотря на более **общие** абстракции, мы **поднимаемся** на уровень выше. Смотря же на более **частные** абстракции, мы **понижаемся** на уровень ниже. Разработчики часто используют понятия *верхнеуровневое* и *низкоуровневое* или *посмотреть сверху на систему* — всё это про уровни абстракции.

## Примеры

Обычно уровни абстракции применяются в контексте *систем*, о которых пойдёт речь чуть позже. Например, архитектура приложения, где приложение — это система, часто имеет разбиение на несколько слоёв, позволяющими посмотреть на приложение на разных "уровнях", или же модель сетевого взаимодействия OSI, описывающая уровни системы работы сети.

Уровни абстракции есть в разных сферах жизни. Например, каждая наука может рассматривать человека по-разному:
- Биологи рассматривают человека начиная с внутриклеточной структуры заканчивая взаимодействием между органами
- Социологи исследуют человека как часть системы общества, исследуя поведение индивидума в социальных взаимодействиях
- Философы смотрят на человека как на существо, имеющее разум, и пытаются понять как этот разум внутри работает и координирует человека в целом
- Программисты могут смотреть на человека как движущийся объект в игре или как систему, по образу которой строится симуляция работы органов

Но абстракция сама по себе не представляет ценности. Именно связи между абстракциями делает изучение абстракций таким ценным. И об этих связях пойдёт речь в странице [Система](system_ru.md).
